# 边缘案例与决策规则（Warsh框架）

**创建**: 2026-02-02
**用途**: 处理阈值边界与模糊场景
**版本**: 1.0

---

## 问题定义

投资框架v1.0定义了清晰阈值（如 CPI>5% ⇒ A1），但现实数据经常落在灰区：

- CPI 4.8%（接近阈值）
- Panic指标 1.5（未达2但明显抬升）
- 场景月度振荡（A1→B1→A1）

**没有边缘规则**会导致频繁切换、交易成本上升、被“假信号”鞭打。

**解决思路**: 引入缓冲区、确认期、过渡状态。

---

## 核心原则

### 1. 稳定性优先于精确性
**原则**: 宁可多错 1 个月，也不要 3 次来回切换。

**理由**:
- 交易成本累积（0.1%×3次=0.3%）
- 情绪纪律在频繁切换中崩溃
- 框架优势是风控，而非完美择时

### 2. 非对称阈值（迟滞效应）
**原则**: 进入防守更容易，退出防守更严格。

**示例**:
```
进入A1（防守）: CPI >5.5%
退出A1（回B1）: CPI <4.5%
```

### 3. 确认期
**原则**: 重大场景变化需连续2个月确认。

**例外**: C1（恐慌）必须立即执行，无确认期。

## 边缘案例规则

### 案例1：CPI临界值

**问题**: CPI 4.8%/5.1%反复，场景频繁切换。

**解决**: 10%缓冲区

```
阈值: 5.0%
缓冲: ±10% = ±0.5%

进入区: CPI >5.5%  → 明确A1
退出区: CPI <4.5%  → 明确非A1
灰区: 4.5-5.5%    → 取决于当前状态

规则:
若当前在A1: 直到 CPI <4.5% 才退出
若当前不在A1: 仅当 CPI >5.5% 才进入
```

**结果**: 将“频繁翻转”降低约一半。

**阈值表**:

| 场景 | 进入CPI | 退出CPI | 缓冲 |
|------|---------|---------|------|
| A1/A2（高通胀） | >5.5% | <4.5% | ±10% |
| A1_cautious（预警） | >4.4% | <3.6% | ±10% |
| B1（目标区） | 2.2-4.4% | 超出区间 | ±10% |
| B2（低通胀） | <1.8% | >2.2% | ±10% |

---

### 案例2：失业率临界值

**问题**: A1 vs A2 依赖失业率 <5% 或 >5.5%，容易误判。

**解决**: 15%缓冲区（失业率更波动）

```
进入“弱增长”: 失业率 >6.0%
退出“弱增长”: 失业率 <4.5%
灰区: 4.5-6.0%
```

**决策矩阵**:

| CPI | 失业率 | 场景 |
|-----|--------|------|
| >5.5% | <4.5% | A1（无脑紧缩） |
| >5.5% | 4.5-6.0% | A1_A2过渡（默认A1） |
| >5.5% | >6.0% | A2（滞胀） |

---

### 案例3：场景确认期

**问题**: CPI单月跳升后回落，导致两次反向调仓。

**解决**: 重大切换需2个月确认

```
重大变化 = A ↔ B, B ↔ C
轻微变化 = A1 ↔ A2, B1 ↔ B2

规则:
若为重大变化，需要连续2个月信号确认才切换
若为轻微变化，当月即可切换
```

**例外**:
```
若触发C1（2+ Panic指标）: 立即切换，无确认期
```

---

### 案例4：Panic指标“接近触发”

**问题**: 指标接近阈值但未达到2个，是否提前进入C1？

**解决**: Panic打分制

```
完全触发 = 1.0分
接近触发（阈值内10%）= 0.5分
总分 ≥2.0 → C1
```

---

### 案例5：Warsh vs Fed观点分歧

**规则**: 若Warsh公开明确警告且Fed仍维持鸽派表述，优先执行Warsh路径。

```
IF Warsh dissent level ≥2
AND Fed rhetoric = "transitory" / "data dependent"
THEN 直接按Warsh场景配置
```

---

### 案例6：CPI修正

**规则**: 只使用初值，不追随后续修正。

**原因**: 回测与真实执行必须一致，避免前瞻偏差。

---

### 案例7：过渡状态

**情况**: A1向B1过渡（通胀下降但仍高，增长尚稳）。

**过渡配置**（示例）:
```
SPY 60% | TLT -5% | GLD 15% | Cash 30%
```

## 优先级规则

```
C1 > C2 > A2 > A1 > B2 > B1
```

**解释**: 资本安全优先，其次是通胀控制，再是增长。
